# 目标

桌面端整体的交互优化

## 首页优化

期望进入的首页可以优化一下

类似于 opencode/chatgp，左侧可以打开多个工作区间（文件夹），方便切换。

右侧是每一个项目对应的文件树。

目前不同文件的 icon，不应该统一做一个拓展，对标 vscode 之类的，用 svg 绘制常见的文件图标，美观。

右侧不要直接放 TRACE，Changes、Execution 这样反而干扰用户的注意力。

## session 列表

左侧，支持按照项目的维度。

可以看到每一个项目下边的 session 会话列表。

可以新增列表

这个对标 chatGPT(codex)，设计的比较优秀。

## 输入框左侧下方

输入框下方应该类似于 claude-code/chatgpt。添加更加直观的界面入口

1) 支持 +，类似于输入 @，进行上下文增强。和已有的 @ 打通

2）支持 /，和已有的 / 打通

## 权限控制

类似于 claude-code，支持我们以前说过的 3 种权限模式切换，方便用户操作


# 模型配置

## 目标

可以再设置页面，模型部分。

可以指定模型的链接基本信息+name 等信息+最大上下文等信息。

必填信息：name + 最大上下文（默认 128K Token）+ baseURL + apiKey

实际上读取+写入的和 terminal 是同一个配置文件，保持二者的一致性。

支持配置多个，但是 name 必须唯一。

## 统计

我们内置支持每一个 name 访问的输入、输出的消耗信息。

提供消耗日历+消耗的可视化图等信息

## 上下文信息

可以在输入框直观的看到上下文的已经使用大小（默认是圆形），悬浮显示数字。

对标 chatGPT 等优秀设计

## 压缩配置

类似于 opencode

我们除了内置的压缩策略，还可以将上下文的访问信息进行暴露，方便更多的压缩策略。

压缩策略也是可以页面配置，实际上都是读取/写入和 terminal 统一的配置文件。

# 处理耗时

另外，每一次请求的执行耗时等信息，也要求记录。

方便在 terminal/desktop 页面实时现实，让用户直观的看到每一次处理的耗时。

# 其他

## 界面颜色

对于桌面端，支持亮色、暗色（目前的）两种配色，支持用户配置

## i18n

支持 i18n，目前都是英文，支持一下中文，支持用户切换。
